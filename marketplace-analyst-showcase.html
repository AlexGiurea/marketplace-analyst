Marketplace Analyst

A web app that pairs a Marketplace®-style bikes-strategy simulation with a grounded AI coach. The coach answers in plain language while staying anchored to the current quarter snapshot: KPIs, brands, competitors, manufacturing, finance, and scorecard themes are compiled into retrievable text chunks, and the model calls tools to pull exact figures from the live data — so it never invents numbers.

Architecture

How a question becomes a grounded answer

The React app posts the scenario + chat turns to /api/chat. The server compiles knowledge, retrieves relevant chunks, then runs an OpenAI Responses loop where the model may call snapshot-backed function tools before composing the final answer with citations and optional widgets.

Same-origin dev on port 5173 Vercel: api/chat.ts + api/health.ts
Orchestration

The chat tool-calling loop

Up to 8 rounds per question. The model is free to call tools until it has the numbers it needs, then composes a grounded answer with citations and, when useful, one or two widgets.

1

Compile knowledge

Active quarter's snapshot → chunks for meta, performance, brands, competitors, scorecard themes, manufacturing, marketing, finance, and HR. Every fact becomes searchable text with an ID.

2

Retrieve chunks

Lexical tokenizer drops stop-words, detects explicit quarter references (Q2, quarter 3), and picks the top-k (≤ 8) chunks most relevant to the user's question.

3

Call the model

System prompt encodes the non-negotiable rules + overview + retrieved chunks. The model may emit function_call items instead of a final message.

4

Tool loop → answer

Server runs each requested tool against the real snapshot and feeds outputs back. When the model stops calling tools, output_text returns plus retrievedChunkIds.

Guardrails

Non-negotiable rules baked into the system prompt

  • Treat the snapshot + tool outputs as the only source of truth for numbers.
  • Never invent KPIs, financial figures, or competitor data.
  • Never make the decision for the student.
  • Frame as evidence, tradeoffs, risks — not recommendations.
  • Plain language, ≤ 120 words by default, few key numbers.
  • 1–2 square-bracket citations per concrete claim.
  • At most one tradeoff widget; up to two chart widgets.
  • Surface missing data — then still answer with what's available.

Function tools

8 tools that return exact figures from the live snapshot

Every tool takes an optional quarterIndex (omit to use the active quarter from the UI). Numeric truth stays in the snapshot — chunks drive routing; tools drive precision.

Overview#1

get_team_quarter_overview

High-level team, quarter, overall share, revenue, and net income.

Brands#2

get_brand_performance

Per-brand demand, sell-through, stockout, pricing, revenue, brand profit. Accepts brandName.

Scorecard#3

get_balanced_scorecard

All Balanced Scorecard themes with scores, priors, trends, weights, cumulative index.

Finance#4

get_finance_and_cash

Debt, dividends, begin/end cash from accounting, interest note.

Manufacturing#5

get_manufacturing_capacity

Capacity, utilization, lost sales, inventory, next-quarter demand forecast, facilities.

Go-to-market#6

get_marketing_and_sales_summary

Marketing spend mix, pricing notes, salespeople, regional coverage.

Trends#7

get_quarter_history_overview

Compact KPI row per quarter — share, revenue, net income, scorecard, ending cash.

Trends#8

get_brand_history

Per-brand, per-quarter series for demand, sold, revenue, and brand profit.

Coach widgets

The coach can reply with more than text

When a visual helps, the model emits fenced coach-widget JSON blocks. The frontend parses them and renders interactive tradeoff comparisons or charts, with inline citations that deep-link to the right workspace panel.

Deployment

One port locally, serverless in prod

  • npm run dev — single Node process serves Vite + /api on port 5173.
  • vercel.json builds frontend/dist and deploys api/chat.ts + api/health.ts.
  • Secrets stay server-side: OPENAI_API_KEY set in Vercel env, never in client bundles.
  • Integration tests hit the real Responses API when OPENAI_API_KEY is configured.
